In this lab, you will learn how to deploy the Red Hat build of Keycloak on a Red Hat OpenShift Container Platform cluster. You will utilize the Keycloak Operator to automate the management and lifecycle of your Keycloak instance.

Before we begin, ensure you have access to a Red Hat OpenShift cluster running version 4.12 or later. You will also need the `oc` command-line tool, authenticated with sufficient privileges to create projects and manage operators, and you should have a designated OpenShift project namespace in mind for this deployment.

Let’s move to our first task: installing the Keycloak Operator.

The Keycloak Operator is essential because it automates the deployment and management processes on OpenShift. To get started, first log in to your OpenShift cluster using the `oc login` command, followed by your username and the cluster API URL. Next, create a dedicated project for your work by running `oc new-project rhbk-lab`. Once the project is ready, you will install the operator from the OperatorHub. Create a subscription manifest using the `operators.coreos.com` API, specifying the stable channel and the `redhat-operators` source. After you have defined this, apply the configuration to your cluster using the `oc apply` command.

Once the operator is running, you can proceed to Task 2: deploying the Keycloak instance itself. To do this, you must define a Keycloak Custom Resource. Create a file named `keycloak-instance.yaml`. Inside this file, you will specify the instance settings, including the number of instances, the container image from the Red Hat registry, your database vendor and host, the hostname for your console access, and the reference to your TLS certificate secret. After saving your configuration, apply it to the cluster by running `oc apply -f keycloak-instance.yaml`.

In Task 3, we will verify the deployment. You need to ensure that the pods are up and running and that the operator has successfully initialized the instance. Check the status of your pods by running `oc get pods -n rhbk-lab`. To confirm that the Keycloak Custom Resource is fully operational, run `oc get keycloak rhbk-instance -n rhbk-lab` and verify that the status is reported as "Ready."

Finally, in Task 4, we will explore the Admin Console. Once the deployment is ready, you will need the initial administrator credentials. You can retrieve the password from the secret automatically created by the operator. Use the command `oc get secret rhbk-instance-initial-admin` with a JSON path filter, and pipe the output to `base64 --decode` to reveal the password. With this password in hand, navigate to the URL you defined in your hostname configuration to access the Admin Console and log in with the username `admin`.

If you encounter any issues during this process, consider these troubleshooting tips. If you see a Pod CrashLoopBackOff status, use `oc logs` to inspect the pod for configuration errors or database connectivity problems. If you encounter image pull errors, double-check that your cluster has access to the Red Hat registry or verify your `ImagePullSecret` configuration. Lastly, always confirm that your version of OpenShift is compatible with the version of the Red Hat build of Keycloak you are deploying.
